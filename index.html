✈ FAST Update Generator — Help

🗂 How it works

  1. Create your profile (top bar) — this is just your name as the engineer using the app. All your chantiers (projects) are stored under your profile in this browser.
  2. Create a project — click "+ New project" or upload an RFS PDF to auto-fill. Each project = one work order / chantier.
  3. Step 3 Condition — add the work items from the RFS. These are fixed for the duration of the chantier.
  4. Step 4 Daily updates — every day, click "+ Today's update". For each work item, select its status and write a free-text description of what was done. You can also add a global free text at the bottom of the day.
  5. Step 6 — add the onsite FAST engineers. Up to 3 engineers can be listed. Save their profiles for reuse across projects.
  6. Generate — click the green button. The table is built automatically in the exact FAST email format.
  7. Copy & paste the table directly into your email client. Copy the subject line separately.
  8. Mark A/C RTS when the chantier is done. The project is archived but stays accessible.

💾 Data storage

All data is stored locally in your browser (localStorage). Nothing is sent to any server. Works offline once opened. If you clear your browser data, your projects will be lost — consider exporting the HTML file as a backup.

🔐 PDF extraction (Gemini)

Sign in with Google (free) to enable auto-extraction from RFS PDFs using Google Gemini. Your Google account is used only to authenticate with the Gemini API — no data is stored.

📱 Install on phone (PWA)

🤖 Android (Chrome)
  1. Open the HTML file in Chrome
  2. Tap the ⋮ menu (top right)
  3. Tap "Add to Home screen"
  4. Confirm — the app appears on your home screen like a native app
🍎 iPhone / iPad (Safari)
  1. Open the HTML file in Safari (not Chrome)
  2. Tap the Share button (box with arrow at bottom)
  3. Scroll down and tap "Add to Home Screen"
  4. Tap Add — the app appears on your home screen

⚠ On iPhone, the file must be served via a web server or opened from a cloud link (not a local file path) for the PWA install option to appear.
